## 1. 제품 핵심 원칙

### 1.1 기본 작동 방식
- **P1 상징** → **P2 신학 주제** → **P3 컨텍스트 보정** → **Top-1 구절+이유**
- "왜 이 구절인가" 2-3문장 근거 + **기도 한 줄** 기록
- **설명 가능** + 예장통합 **교리 가이드라인** 내재화

### 1.2 경험 원칙 (One Scene, One Verse)
- 한 장의 사진에서 **핵심 상징 1-2개** 추출
- **한 절(또는 두 절)** 제시
- 대안 구절(ALT-1)은 사용자가 직접 선택할 때만 노출

## 2. 지양점 (Out of Scope) - 절대 금지 사항

| 카테고리 | 지양 기준 |
|---|---|
| **심판/경고 단독** | 복음·소망 맥락 없이 심판·경고 구절만 단독 노출하지 않는다 |
| **재난/사고 공포** | 재난·사고 사진에는 공포 증폭이 아닌 **위로·평강 계열 구절을 우선 매칭**한다 |
| **정치·갈등 선동** | 정치·이념 갈등을 선동하는 방향의 매칭과 워딩을 피하고, 중립적 표현을 유지한다 |
| **예언/점괘식 단정** | 특정 상황을 미래 예언·점괘처럼 단정하는 표현을 피하고, 도구는 조력자로만 작동하도록 명시한다 |
| **민감 대상 공개** | 아동·의료·장례 등 민감 장면은 기본값 비공개·익명 권고를 한다 |

## 3. 신학적 전제 및 가치 선언

1. 이 앱은 **예언·점괘식 단정**을 지양하며, 사용자의 상황을 미래에 대한 단정적 선언으로 해석하지 않는다
2. 모든 추천은 **본문 문맥을 존중**하며, 단어 일부만 발췌해 의미를 왜곡하지 않는다
3. **복음과 소망의 균형**을 유지하며, 심판·경고 구절도 복음·회복 문맥 안에서만 사용한다
4. 온라인 환경에서도 **공동체 윤리**(존중, 배려, 비폭력)를 중시하며, 혐오·공격적 사용을 제한한다
5. 이 앱의 추천은 **목회·신앙 결정의 보조 수단**일 뿐, 인간 목회·상담을 대체하지 않는다는 점을 명시한다

## 4. 교리 기준 (예장통합 가이드라인)

### 4.1 기본 교리 틀
- 개신교 **장로교(개혁주의) 전통**
- **사도신경**, **대한예수교장로회 12신조**
- **웨스트민스터 신앙고백**과 **대·소요리문답**

### 4.2 핵심 전제
- **삼위일체 하나님**
- **성경의 영감·무오성과 최종 권위**
- **은혜에 의한 구원**
- **교회 공동체의 중요성**

### 4.3 구속사 중심 해석
- 개별 구절은 성경 전체의 **구속사적 흐름(창조–타락–구속–완성)** 안에서 해석
- 해당 본문이 **그리스도 안에서 어떻게 성취·완성되는지**를 설명 카드에서 다룸

### 4.4 본문 문맥 존중 원칙
- **문맥을 벗어난 키워드 매칭**을 지양
- AI가 구절을 추천할 때, 해당 **장·단락 전체 메시지와 어긋나지 않는지** 점검
- 각 구절에는 "**허용 / 주의 / 금지**" 태그 부여 가능

### 4.5 사랑/공의 · 개인/공동체 균형
- **개인 심리 위로**에만 편향되거나, 반대로 **정죄·심판만 강조**하지 않도록
- **사랑과 공의, 위로와 회개, 개인과 공동체의 균형**을 지향

## 5. 민감 주제 처리 원칙 (핵심)

### 5.1 고난·죽음·장례·우울·자살
- ❌ 단순한 "이겨내라" 식 구절 사용 금지
- ✅ **하나님의 동행·위로·눈물·소망**을 중심으로 한 본문을 우선 사용
- ✅ 필요 시 **전문 상담/도움 안내 문구**를 함께 표기

**우선 주제:**
- 평강·샬롬
- 보호·인도
- 치유·회복
- 사랑(아가페)
- 소망·부활

**억제 주제:**
- 심판·경고
- 회개 (단독 사용 지양)

### 5.2 성윤리·가정 폭력·혐오 이슈
- 성경적 윤리를 정직하게 드러내되
- 특정 소수자·집단에 대한 **혐오 조장 문맥**으로 오해될 여지가 없도록 언어와 문맥을 조정

### 5.3 정치·전쟁·국가 갈등
- 특정 정당·이념·국가 선전처럼 읽히지 않도록
- 가능하면 **복음·평화·정의·화해 관점의 본문** 중심으로 제한

## 7. AI 활용 신학·윤리 가이드라인

### 7.1 AI의 역할 정의
- AI는 **새 계시를 주는 존재가 아니라**, 이미 주어진 **성경 본문과 교회 전통**을 바탕으로
- 사용자가 현재 상황에서 **묵상할 말씀을 찾도록 돕는 도구**로 한정

### 7.2 권위의 우선순위
- 최종 권위는 언제나 **성경 본문**과 **교회 공동체(목회자·장로회·공동 체험)**에 있음
- AI 추천은 언제나 **"제안"**이며, 사용자가 **받아들이거나 넘어갈 자유**를 전제

### 7.3 허용되는 기능 / 허용되지 않는 기능

**✅ 허용되는 기능:**
- 사진·텍스트·상황 정보를 기반으로 **해당 상황과 잘 맞는 성경 구절 추천**
- 구절의 **기본 설명·배경·적용 포인트 제공**
- **연관 구절·묵상 질문 제안**

**❌ 허용되지 않는 기능:**
- 성경 본문을 **수정/삭제/추가**하는 행위
- "**하나님이 당신에게 이렇게 말씀하셨다**" 식의 **직접 계시 문구 생성**
- 특정 인물·집단에 대한 **정치·경제적 의사결정(투표, 투자, 전쟁 찬성/반대 등)**을 직접 지시하는 출력

### 7.4 AI 오·남용에 대한 보호막
- 사용자가 AI 출력을 "**내가 이 결정을 해야 한다는 하나님의 직접 명령**"으로 오해할 위험이 있는 부분에는
  - "**이 앱의 추천은 목회적·전문적 상담을 대체하지 않습니다**" 유형의 안내 문구를 배치
- 극단적·위험한 상황(자살 암시, 폭력 계획 등)이 감지될 경우
  - 특정 구절 추천만으로 끝내지 않고
  - **위기 대응 안내(상담 전화, 도움 요청 권장 문구)**를 함께 노출

## 8. 실무 적용 규칙 (코드 구현용)

### 8.1 금지 조합 (즉시 차단)
1. **병상/침대 + 심판·경고** → 차단
2. **밤/어둠/자정 + 심판·경고** → 차단
3. **고난 상징(눈물, 고통) + 회개 단독** → 균형 필요 (위로 주제 함께 제시)

### 8.2 우선 주제 (컨텍스트별)

**고난 상황 (병원, 밤, 침대, 눈물, 고통):**
- 우선: 평강·샬롬, 보호·인도, 치유·회복, 사랑(아가페), 소망·부활
- 억제: 심판·경고, 회개

**죽음·장례 상황:**
- 우선: 소망·부활, 평강·샬롬, 사랑(아가페), 보호·인도
- 억제: 심판·경고

**재난·사고 상황:**
- 우선: 보호·인도, 평강·샬롬, 치유·회복
- 억제: 심판·경고

### 8.3 컨텍스트 보정 계수

**시간대:**
- 새벽: 기도(×1.3), 소망·부활(×1.3), 말씀·진리(×1.3)
- 밤/자정: 평강·샬롬(×1.3), 보호·인도(×1.3), 심판·경고(×0.2)

**장소:**
- 병원: 치유·회복(×1.5), 평강·샬롬(×1.5), 심판·경고(×0.3), 회개(×0.3)
- 교회: 예배·찬양(×1.3), 공동체·연합(×1.3), 말씀·진리(×1.3)
- 장례식장: 소망·부활(×1.5), 평강·샬롬(×1.5), 심판·경고(×0.1)

### 8.4 주제 안전도 등급 (S3 점수 기준)

**안전 (0.9):** 사랑(아가페), 평강·샬롬, 소망·부활, 은혜, 보호·인도, 치유·회복, 믿음·신뢰

**중립 (0.7):** 창조·섭리, 임재·영광, 거룩·성화, 기도, 말씀·진리, 예배·찬양, 순종·겸손, 인내·시험, 지혜·분별, 성령·열매, 공동체·연합, 소명·일·청지기, 가정·양육

**주의 (0.6):** 회개, 정의·공의, 화해·용서(관계)
- 단독 사용 지양, 복음·소망 문맥 필요

**위험 (0.4):** 심판·경고
- 복음 문맥 없이 단독 사용 금지

### 8.5 페널티 규칙 (S4)

| 조합 | 페널티 | 이유 |
|---|---|---|
| 침대/병상 + 심판·경고/회개 | -0.8 | 병상에서 심판·회개는 부적절 |
| 밤/자정 + 심판·경고 | -0.6 | 밤 + 심판은 공포 증폭 |
| 눈물/고통 + 회개/정의·공의 | -0.5 | 고난 중 정죄는 2차 상처 |

## 10. 데이터·프라이버시 윤리

### 10.1 민감 이미지 처리
- 얼굴, 어린이, 민감한 장면 등이 포함된 사진 처리 시:
  - **비식별화**
  - **최소 데이터 보존**
  - **민감 이미지 자동 블러/차단**

### 10.2 개인 데이터 보호
- 사용자의 기도 내용·감정 기록은 **로컬 우선 저장(단말 내)** 기본
- 클라우드 저장 시 **강력한 암호화**와 **최소 권한 원칙** 적용

### 10.3 위치 정보
- GPS 정밀 좌표는 수집하지 않음
- **광역 단위로만 집계** (지오해시 6-7자 수준)

## 요약: AI 개발자가 반드시 기억해야 할 3가지

1. **절대 금지**: 병상/밤 + 심판·경고, 예언·점괘식 단정, 본문 왜곡
2. **고난 상황 우선**: 평강·샬롬, 보호·인도, 치유·회복, 사랑, 소망·부활
3. **균형 유지**: 복음과 소망의 균형, 사랑과 공의의 균형, 개인과 공동체의 균형
